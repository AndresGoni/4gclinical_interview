The first valid command to the robot is a PLACE command, after that, any sequence of commands may be issued, in any order, including another PLACE command.

The application should discard all commands in the sequence until a valid PLACE command has been executed.

X and Y values are entered manually. They must be an integer value between 0 and 4 to fit the game board dimensions.

The robot will be oriented to the direction chosen in the list (NORTH, SOUTH, EAST or WEST)

Game commands

MOVE will move the toy robot one unit forward in the direction it is currently facing.

LEFT and RIGHT will rotate the robot 90 degrees in the specified direction without changing the position of the robot.

REPORT will announce the X,Y and F of the robot. This can be in any form, but standard output is sufficient.

The toy robot must not fall off the table during movement. This also includes the initial placement of the toy robot.

Any move that would cause the robot to fall must be ignored.
